Context

Built to make
concepts stick

This project transforms structured technical notes into an interactive study guide. Rather than keeping reference material scattered across documents, the content was rebuilt as a single web toolkit — designed around active recall instead of passive reading.

The toolkit covers the full BI analytics stack: from Star Schema data modeling and DAX measure design to domain-specific KPIs for Finance and HR functions, plus an overview of the modern data engineering ecosystem.

Published as a public, forkable reference — any BI practitioner can adapt it for their own study process or use it as a professional reference on the job.

Open resource

Not affiliated with any company. All content is intended for educational and professional development purposes. Fork the repo, add your own topics, and adapt it for your own workflow.

Scope

Three files, full
BI coverage

The toolkit is split into three focused files: a main reference guide covering all six topics, a DAX deep-dive with 12 exercises and a 30+ function cheatsheet, and an interactive quiz with 35 questions across all topic areas.

The main guide covers Star Schema design, SQL patterns, KPIs for Finance and HR, and an overview of the modern data stack. It's structured as a living reference — practical, scannable, and built to be used on the job, not just studied once.

The quiz gives immediate feedback after each answer, making it useful for self-assessment before interviews or for reinforcing recently studied concepts.

Knowledge Areas

Six technical tracks

Each track covers core concepts, practical patterns, and real-world applications across the BI analytics stack.

Star Schema & Data Modeling
Core data modeling concepts — fact tables, dimension tables, grain definition, and schema design patterns used in BI environments.
  • Fact vs dimension tables
  • Star, Snowflake & Galaxy schemas
  • Grain definition
  • Keys & relationships
DAX
Deep-dive into DAX language patterns — from basic aggregations to time intelligence and advanced filter context manipulation.
  • CALCULATE, ALL & FILTER patterns
  • Time intelligence (YTD, MTD, YoY)
  • Measures vs calculated columns
  • VAR, DIVIDE & SWITCH
SQL
Advanced SQL patterns for BI analytics — window functions, CTEs, complex joins, and data quality checks used in real-world reporting workflows.
  • Window functions (RANK, LAG, LEAD)
  • CTEs and subquery optimisation
  • JOIN types and patterns
  • Data quality check patterns
KPIs Finance
Finance domain KPIs used in reporting and FP&A — from P&L metrics and budget variance analysis to forecasting accuracy.
  • EBITDA & operating margin
  • Budget vs Actual variance
  • Forecast accuracy
  • Cash flow metrics
KPIs HR / People Analytics
HR and People Analytics metrics — workforce KPIs, talent acquisition indicators, engagement measures, and workforce planning fundamentals.
  • Turnover & retention rates
  • Headcount & FTE planning
  • eNPS & engagement
  • Time-to-fill & absenteeism
Data Stack
Overview of modern BI and data engineering tools — from Power BI and cloud storage to orchestration platforms and HRMS integrations.
  • Power BI & DAX engine
  • AWS (S3, Glue, Athena)
  • HRMS tools
  • Jira & project tracking
Design

Why interactive?

Passive reading doesn't make concepts stick. Active recall does.

Active recall over passive notes
Every guide is paired with quiz questions and exercises. You don't just read about CALCULATE — you write it, debug it, and explain it back. The format forces retrieval, not just recognition.
Finance AND HR domains covered
Most BI reference materials focus only on technical content. This toolkit also covers domain-specific KPIs for Finance and HR — the two most common BI analyst verticals in the market.
Forkable and adaptable
Pure HTML — no framework, no dependencies, no build step. Clone the repo, add your own topics or domain KPIs, and you have a customised reference in minutes.
Works offline, no setup
No CDN calls, no API, no build step. Open the file in any browser and it works — on a plane, in a hotel, anywhere. Progress is stored in localStorage between sessions.
Technology

Zero dependencies

Built entirely with vanilla HTML, CSS, and JavaScript. No framework, no CDN, no installation. Open in a browser and it works.
